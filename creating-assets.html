Creating assets

Seb Jensen

I have collected a number of assets from the Rocket League Esports website that I am looking to rework and adapt for a different audience in a kiosk format. However, there are a number of additional assets that I will need to create to accompany what I have collected. I can use what I have taken from the source and adapt them by harnessing various media production techniques. This will involve the use of Photoshop mostly, though PowerPoint also offers limited options for adding borders, recolouring images and such, too.

When creating and adapting these assets, I will follow my already established means of organising the files. Except, all of my new files will fall under a “Adapted / Original” folder at the top level, alongside the “Source” folder that I was using last session. I will ensure that I am using appropriate file formats, such as using .PNG where transparency is necessary and using .JPG for smaller files where it is not. I will also keep up with giving my new assets sensical names and placing them in a similar file structure. This will make my assets folder easily navigable and searching across all folders for particular assets will be easy, too.

By using Google Drive for Desktop, I am able to access all of my files regardless of which device I use each session. My adapted and original assets will also be synced with the cloud for ease of use and access.

Start

I want to include a splash screen that will display at the start of the kiosk experience when it isn’t being used. I detailed this in my post about structure and layout. I have used the font identified when conducting my focused research and the colours, too.

To create the button, I used the shape tool in Photoshop to create a parallelogram and apply a gradient colour over the top. I then used the text tool to add the “Start” text, intentionally using bold weighting to emphasise it.

Homepage

I wanted the featured buttons on the homepage to be eye-catching, and started first and foremost by creating a yellow-gold stroke. This draws the user’s attention to the content, and I have used the same font used by RLCS for the caption to ensure continuity and consistency. I have made use of two of the assets I collected last session.

Menu & Navigation

I wanted to create a selection of bold and prominent button designs to be used in a single menu and navigation slide. The size of each button establishes hierarchy, and some also use the assets I collected last session to further engage the user. To accomplish this, I had to use gradients for where I wanted the colour to fade out, and layers to display the text above the background colour. There was also the application of a faint drop shadow with the text to create distance and a sense of depth between layers.

Background

All backgrounds will follow a consistent design. There should be a half-opaque bar in the top half of the background where the title will be placed. This will increase legibility, as I wish to use the same orange-gold colour for my text. I used the assets from the source I collected and used layers to reduce the opacity of a black rectangle I created with the shape tool.

Fan Rewards

I used text, gradients to increase legibility, assets that I collected from the source, and layers to organise all of these. The differing sizes in text establishes hierarchy. I wanted to display some of the fan rewards that the user could obtain for watching the tournaments live. These will all be displayed on a fan rewards slide.

Logos & Extras

I added a drop shadow to the logo to create a sense of depth.

I also took social media platform logos from the Internet and applied them to an orange-gold shape that I created beforehand with the shape tool.

Perspective project approach

The main strength for this session was my creation of a multitude of new assets, and how I’ve been able to draw from source assets I had collected and features I had identified in my focused research. I will be able to combine all these assets into a final product for my response to the brief.

It would be even better if I had created a few more assets from scratch as opposed to drawing from the assets I had already collected as often as I did. This would consolidate my efforts to adapt the existing website for a different audience in a kiosk format. While I have still managed to do so, making more assets entirely independently would further benefit me in reimagining an existing product.

My short-term goal is to look at how I can use the master slide. I already have thoughts about how I could use its functionality to ensure a consistent house style throughout my product. This includes colours, fonts, and how I can use the same background with every slide.

My long-term goal still looks to how I can combine my newly created and adapted assets with the source assets that I collected last session. To accomplish this, I will also need to consider animations and the various interactive features that I wish to incorporate into my product.
