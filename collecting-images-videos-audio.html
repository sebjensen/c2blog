Collecting images, videos, audio

Seb Jensen

The proper organisation of collected assets in a sensical folder structure is quintessential to an efficient workflow. In today's session, I have collected all the images, videos, and audio that I will need for me to rework the Rocket League Esports website in response to the brief. In my production documents folder, I have created a 'Source' sub-folder, which then branches into an intuitive setup with all the assets I have collected. At the top layer, I will also be able to make another folder for any adapted and newly created assets.

It is also important to ensure that all files and collected assets are named effectively and appropriately. I have ensured that all file names offer the necessary detail needed to understand its purpose and contents with or without a file preview. For example, "FR Instigator Title.png" makes it clear that the image is related to fan rewards ("FR"), and is the Instigator Title cosmetic in-game.

I have also considered file extensions and formats, balancing the need for alpha data in some assets (such as to allow transparency in a logo) and file size optimisation (where a .JPG might be used).

I am utilising Google Drive for Desktop to keep my files and data in sync with the cloud. This means that I will always have access to my files online, regardless of which device I am using.

Perspective project approach

This session's strengths included my thorough consideration of file management and navigation when collecting and storing assets from the source (Rocket League Esports website). The setup is intuitive, and it is also stored in and synced with the cloud, so I can access it from any device in school.

It would have been even better if I was able to find individual .mp3 files for the in-game goal event chimes. While I still have all the audio, it is just in a single file, and I will need to cut through this and trim it down to the specific sound effect I want in the future. Having each chime sound effect separate would be more sensical.

Now that I have collected images, video, and audio, I can begin to look at creating my own assets in the next session as a short-term goal. I will also be able to adapt what I have collected and will also document this.

In the long-term, I will have to think about how I can incorporate the various interactive elements and features that I outlined in my initial research document. This included the use of buttons, galleries, and the inclusion of a video. I will be able to use the assets I have collected today for this.
